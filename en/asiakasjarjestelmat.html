Customer Systems

The application's data can be utilized in various customer systems. For example, a company can export employee stamping data to Excel and use it for payroll calculation. In the future, the goal is to publish the program with its source code, allowing stamping data to be retrieved directly via an API. In connection with this, it is also possible to implement the maintenance of stamper and stamping type data through the API.

The implementation of GPS location allows, for example, competition organizers to have information about competitors' locations and to implement a real-time map display on the internet. One variation I have implemented is tracking a sailing competition on a map and calculating the results. However, this requires the activation of the competition setting. In that case, start and finish lines, as well as intermediate points, must also be added. Stamps are recorded in real-time (at intervals of 1 second to 1 minute) between the start and end stamps.

Sample programs for utilizing the application's API

The first program has now been implemented. The program is implemented in the RUST language and can be downloaded from GitHub at the Aikaleima-Console repository and compiled yourself. I use a Windows environment myself, but the programs are command-line based and platform-independent, so they can be compiled for your desired environment.

Returnkey

The program returns a login key that every application will use in the future. For API use, a user must be created in user management. A one-time key is given as a parameter to the program, and the API call then returns a permanent key.
