Timestamp Service

Timestamp is a service that allows a customer to manage various stamping events. The core parts of the system are the management of stamping data and the stamping terminal. The system is customer-specific and can be found at customer.aikaleima.fi. Only the customer's users and the service administrator can access the data. The data is located in a customer-specific database and, for stamping data, only until it has been transferred to the customer's systems. After the transfer, the customer can delete the data from the system.

The system/software, whatever you want to call it, is a so-called Proof of Concept. Not a commercial version. With this system, I aim to test how a Blazor Server Side web application works in a production-like environment. And how maintenance and duplicating the application for multiple customers can be done as simply as possible. If you are a programmer and have struggled with the same things or are just interested in the code and/or concept, please get in touch. Those who have struggled with AWS and Azure will surely agree that there must be a simpler and cheaper way to implement production systems.
